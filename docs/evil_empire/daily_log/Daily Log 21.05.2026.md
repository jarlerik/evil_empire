# Daily Log 21.05.2026

## fix(mobile): preserve reschedule state across week navigation
Rescheduling a missed workout to a different week was broken — the pending move was cleared both by the week prev/next handlers and by an existence check that fired whenever the moved workout wasn't in the currently loaded week's data. Carry the workout/session name on `PendingMove` so the banner survives the week change, and only clear the state on explicit cancel or delete.

## feat(programs): "Skip and push" — cadence-aware program slip
Missing one session in a progressive program (e.g. 2×/week Russian back squat) should push every following unstarted session forward by one cadence slot — not pile them onto the next scheduled day. Added `programs.slip_slots` (NOT NULL DEFAULT 0) and a cadence-aware slot math layer (`deriveCadence`, `slotIndexFor`, `slotToWeekDay`, `resolveSessionDateWithSlip`, `resolveSessionDates`) in `programScheduling.ts`. `fetchProgramSessionsForDateRange` switched to session-driven iteration so it can honor slip per-session. New `incrementProgramSlip` service + `skipProgramSlot` on the mobile context, wired to a new Skip-and-push button on `ProgramSessionCard` (confirmed via `Alert.alert`). Mirrored on web via `useSkipProgramSlot` and a button in `VirtualProgramSessionCard`. Materialized workouts are unaffected — only virtual session calendar dates shift; the W#/D# label keeps its program-plan identity. Plan in `docs/evil_empire/peakTrack/plans/program-slip-plan.md`.

## fix(mobile): refresh program sessions after Skip-and-push
The mobile screen holds `programSessions` in local state populated once via `loadData`. Clearing the context's session cache wasn't enough — the screen had to re-mount before the slipped layout appeared. Added an `onSkipped` callback to `ProgramSessionCard` that the home screen wires to `loadData({ showLoader: false })`. Web was already covered: `useSkipProgramSlot` invalidates `programKeys.all`, which is a prefix of `sessionsForRange`, so TanStack Query auto-refetches.
